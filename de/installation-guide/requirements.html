Voraussetzungen

Anbieterseite

Zur Installation von TEKSI-Abwasser ist Folgendes erforderlich:

  • Eine PostgreSQL-Datenbank mit PostGIS-Erweiterung (unterstützte Versionen finden Sie in der Roadmap <https://github.com/teksi/Home/wiki/TEKSI-modules-releases-and-roadmap>).

Bemerkung

TEKSI Abwasser verwendet eine PostgreSQL-Datenbank mit PostGIS-Erweiterung. Sie müssen diese auf Ihrem Server installieren: Es wird empfohlen, die EnterpriseDB <https://www.enterprisedb.com/downloads/postgres-postgresql-downloads> zu verwenden, um sicherzustellen, dass Ihre PostgreSQL-Version mit Ihrer PostGIS-Version übereinstimmt und am richtigen Ort installiert ist.

  • Python 3.12+ (installiert via OSGEO4W oder mit python.org)

  • QGIS installiert mit der neuesten LTR-Version (LongTimeRelease)

    Es ist empfohlen, direkt mit dem OSGEO4W-Installer und allen aufgelisteten Python-Paketen zu installieren.

  • Eine funktionierende pg_service-Datei und eine sichere Verbindung zu Ihrem Datenbankserver (gehostet oder lokal).

  • Das TEKSI Module Management Tool (TMMT-Plugin <https://plugins.qgis.org/plugins/teksi_module_management_tool>) installiert aus dem QGIS-Plugin-Repository.

Folgende Python-Pakete (installiert über OSGEO4W oder mit pip) werden benötigt:

  • Psycopg as database adapter

  • Pydantic for data validation

  • Pirogue für Modellerstellung und -aktualisierungen

Bemerkung

Das PUM-Python-Paket wird automatisch mit dem TMMT-Plugin installiert.

Clientseite

Für jeden Client (Arbeitsplatz) benötigen Sie:

  • QGIS installiert mit der neuesten LTR-Version (LongTimeRelease)

    Es ist empfohlen, direkt mit dem OSGEO4W-Installer und allen aufgelisteten Python-Paketen zu installieren.

  • Eine funktionierende pg_service-Datei und eine sichere Verbindung zu Ihrem Datenbankserver (gehostet oder lokal).

  • TEKSI Wastewater plugin (.zip file)

  • TEKSI Wastewater project (.qgs file)

Folgende Python-Pakete (installiert über OSGEO4W oder mit pip) werden benötigt:

  • Psycopg as database adapter

  • sqlalchemy für INTERLIS Im- und Exporte

  • geoalchemy2 für INTERLIS Im- und Exporte

PostgreSQL Server seitig :

Erlauben Sie die Verbindung Ihrer Clients über die Datei postgresql.conf und stellen Sie sicher, dass Ihr Client den PostgreSQL-Server-Port erreichen darf (Standard ist 5432).

TEKSI Abwasser benötigt PostGIS-Funktionen, die in einer Reihe von Releases nicht ordnungsgemäß funktionierten:

  • Versionen vor PostGIS 3.2 sind nicht betroffen.

  • Bei Verwendung von PostGIS 3.2 wurde der Fehler in Version 3.2.7 behoben.

  • Bei Verwendung von PostGIS 3.3 wurde der Fehler in Version 3.3.6 behoben.

  • Bei Verwendung von PostGIS 3.4 wurde der Fehler in Version 3.4.2 behoben.

  • Versionen PostGIS 3.5 sind nicht betroffen.
