2025.0.X

Fonctionnalités principales

  • First version of TEKSI wastewater module for QGIS 3.40 LTR and initialized via the PostgreSQL Update Manager (PUM) and the TEKSI Module Management Tool (TMMT). Check the installation guide for more information on how to install TEKSI wastewater module via TMMT.

note::

Check the new `release video https://vimeo.com/1149117501?fl=pl&fe=cm>`__ to see how these features support “Clean waters and an environment worth living in!”

  • Improved user interface for better usability and accessibility.

  • New styles and symbology

  • Project colors for styles

    More …

  • Extended translation support for French and German

  • Demo data initialization via TMMT

  • Improved checks for INTERLIS Export

  • INTERLIS Export option to export extra organisations

  • New layer configurations for GEP themes : measures events, log cards and rural

  • New documentation on adding log cards

  • And a lot of details added in the documentation. If you like a new feature please add it here!

Installation

  • As this release is published after the development of the TEKSI Module Management Tool plugin (TMMT), check the

installation guide for more information on how to install TEKSI wastewater module via TMMT.

Migration from 2024.X to 2025.X

  • As this release introduces a new database structure, a migration from 2024.X to 2025.X is required.

    A simple way to migrate your data is to export it from the 2024.X database in INTERLIS format.

Note

Please install the 2025.X TWW Plugin in a separate QGIS profile and use the 2025.X Release instead of the latest 2024.X release to export the data to make sure, you get the complete VSA-DSS dataset. Release 2024.X of the plugin does have missing attributes that have been introduced with Release 2020 - a patch for this was only added in 20205.

Note

In future this can be done directly within the database when the TMMT plugin will also support upgrading, not only installation of the database.

and import it into a new empty 2025.X database (also use the latest 2025.X release for that).

Do not forget to migrate your oid-prefix, your default-values for dataowner and provider and to adjust the sequences for correct obj_id’s.

In the future, TMMT will propose updates directly from the changelogs provided in the releases and should be able to handle upgrades with no migration required.

Bug fixes and performance improvements

  • Various bug fixes for SWMM export

  • Improved tests

  • Improved hints for classes, attributes and values
